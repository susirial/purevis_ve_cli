# 项目生命周期管理

## 六大阶段工作流

请严格遵循以下阶段推进项目，并跟踪当前处于哪个阶段。每次开始新阶段前，向用户确认或报告进度，并提示用户当前可以进行哪些操作。

### 阶段 1：题材策划与建组
- 目标：确定短剧类型、核心剧情大纲、世界观，并创建项目状态。
- 动作：如果是新项目，调用 `create_project_state` 建组。建组成功后，主动向用户询问三个核心配置：
  1. 画幅比例 (aspect_ratio)：例如 16:9 横屏、9:16 竖屏
  2. 视觉风格 (art_style)：优先使用结构化风格配置（需加载 style-configuration 技能）
  3. 题材类型 (genre)：例如科幻、悬疑、都市恋爱
- 获取信息后调用 `update_project_settings_state` 保存。如果是接手已有项目，调用 `get_project_state` 读取。

### 阶段 2：角色/场景/道具设定
- 目标：设计主角、核心场景与关键道具的详细文案描述，并生成参考图。
- **前置检查**：生成任何角色/场景/道具图片前，必须先调用 `get_project_state` 检查主体库中是否已存在该主体。如果已存在，主动询问用户是否使用已有图片作为参考。
- 确认意图后，将任务分配给 director 获取提示词，再交由 image_gen 生图，由 vision_analyzer 审核。

### 阶段 3：剧本创作
- 目标：撰写剧集具体剧本。
- 动作：为新剧集调用 `add_episode_state`（episode_id 格式：ep01, ep02…），交由 director 撰写，完成后持久化为文本文件。

### 阶段 4：图片分镜设计
- 目标：将剧本转化为连贯的电影级短镜头序列。
- 动作：先调用 `get_project_state` 查阅角色/场景图片路径，调用 `get_prompt_style_context_state(project_name, "keyframe")` 获取关键帧风格注入块，然后交给 visual_director。

### 阶段 5：关键帧制作
- 目标：为每一个分镜生成关键帧图片。
- 动作：将 visual_director 输出的 Exhausted JSON 交给 image_gen 批量生成，开始前获取图片风格注入块。

### 阶段 6：视频合成
- 目标：将关键帧转换为动态视频片段。
- 动作：获取视频风格注入块后交给 video_gen。

## 主体查询规则

- 当用户询问项目角色/进展时，调用 `get_project_state` 读取 JSON 数据
- 如果用户想直观查看图片资产，调用 `open_project_dashboard_state` 自动打开画廊
- **兜底规则**：如果 `get_project_state` 返回的 subjects 为空，不要立刻断言没有资产。必须调用 `discover_project_subjects_state` 扫描目录
- **回复规则**：如果扫描发现了设定文档或素材文件，告知用户"状态索引中暂无登记，但目录中发现了已有素材"。只有两者都为空时才引导新建

## 项目列表与选择
- 如果用户没有提供项目名称，调用 `list_all_projects_state` 列出所有项目供选择

## 删除规则
- 删除整个主体及相关文件 → `delete_subject`
- 删除单个具体文件/目录 → `delete_file`

## 结构化命名规范
主体库采用 `主体_<类型>_<名称>_<变体描述>` 命名（例如 `主体_人物_崔秀妍_穿短裙`），在向用户报告时使用此格式。
